1 Introduction to JSON

JSON (JavaScript Object Notation) is a plain-text data file format. It was originally intended for use with JavaScript, as the full name implies, as it’s structured to store attribute-value pairs. As a result, it can be used to store data records containing nested and potentially complicated item fields. It is easy for humans to read and write. It is easy for machines to parse and generate. JSON is a text format that is completely language independent but uses conventions that are familiar to programmers of the C-family of languages, including C, C++, C#, Java, JavaScript, Perl, Python, and many others. These properties make JSON an ideal data-interchange language. [Ref]

JSON is built on two parts:keys and values. Together they make a key/value pair.

Key: A key is always a string enclosed in quotation marks.
Value: A value can be a string, number, boolean expression, array, or object.

A key value pair follows a specific syntax, with the key followed by a colon followed by the value. Key/value pairs are comma separated.[Ref]

In JSON, values must be one of the following data types:
a string
a number
an object (JSON object)
an array
a boolean
null

JSON values cannot be one of the following data types:
a function
a date
undefined [Ref]

2 JSON Packages in R

As one would expect with a popular file format, multiple methods exist for reading JSON into R. The rjson library is a very minimal package which doesn’t have any functionality beyond reading JSON data into R (it won’t even convert the contents to a data frame by itself). It was first implemented for R in 2007 by Alex Couture-Beil. It allows R users to convert JSON objects into R objects and vice-verse. There are three basic functions available in the rjson package: fromJSON, toJSON and newJSONParser.

Two other packages RJSONIO and jsonlite were introduced to R community since 2010 and 2013. They could be used as an alternative to the rjson package, but have some advantages thanrjson. For example, RJSONIO package uses a C++ library - libjson, rather than implementing yet another JSON parser so that parsing would be faster than the rjson package. Compared with the two others, the jsonlite package is more powerful. It provides more options in the fromJSON/toJSON functions, which can convert data between JSON and R objects even through a http(s) connection, a pipe, even from a NoSQL database. In addtion, it can also interact with web APIs, building pipelines and streaming data between R and JSON. There is webpages provides a detailed comparison between three packages: A biased comparsion of JSON packages in R. [Ref]

For this vignette, we’ll use jsonlite for the above comparison. More usage about jsonlite could be found at the CRAN page and documentation here).

3 Working with JSON file in R

3.1 About the used JSON file

The dataset used is taken from the CORGIS (Collection of Really Great, Interesting, Situated Datasets) project, a collection of moderately sized datasets intended for experimentation and learning how to handle data operations. It is about cars and how much fuel they use. Four types of information were included: “Engine Information”, “Identification”, “Dimensions”, “Fuel Information”.

The specific dataset can be found here.

3.2 Parse the JSON file to a R object

Reading in a file is fairly simple:

library(tidyverse) 
library(jsonlite) 
json_data <- fromJSON("https://think.cs.vt.edu/corgis/json/cars/cars.json")
class(json_data)
## [1] "data.frame"

The function name is actually the same for all three packages mentioned before. However, the rjson and RJSONIO functions return a nested list, which would require some extra processing, so we stick with this for simplicity’s sake.

So this function apparently returns a data frame. However, if you try to look at the columns:

# try to get the class of each column, but...
sapply(json_data, class)
## Engine Information     Identification         Dimensions 
##       "data.frame"       "data.frame"       "data.frame" 
##   Fuel Information 
##       "data.frame"

The actual object is actually a collection of data frames (which was also called “nested data frame”“). We need to conver such a”nested data frame“” into a regular 2-dimensional regular dataframe. This is where the flatten() function in jsonlite comes in:

# also convert to a tibble to make printing neater
actual_df <- tbl_df(jsonlite::flatten(json_data))
